# Реалізація алгоритмів підрахунку піків для визначення ритму дихання за даними акселерометра

> Інтерактивний Python-застосунок для поетапної обробки сигналів акселерометра та визначення частоти дихання методом підрахунку піків.

## Загальна інформація

- **Тип проєкту**: Python-застосунок з інтерактивним вебінтерфейсом
- **Мова програмування**: Python
- **Фреймворк для інтерфейсу**: Streamlit
- **Бібліотеки**: Pandas, NumPy, SciPy, Matplotlib
- **Призначення**: обробка сигналів акселерометра та визначення частоти дихання методом підрахунку піків

---

## Опис функціоналу

- Зчитування CSV-файлу з даними акселерометра
- Перевірка структури вхідних даних
- Побудова сирих сигналів акселерометра по осях X, Y, Z
- Центрування сигналів для усунення базового зміщення
- Ресемплінг сигналів до частоти 50 Гц для отримання рівномірного часового кроку
- Смугова фільтрація сигналів у діапазоні дихальних коливань
- Пошук піків на відфільтрованому сигналі
- Розрахунок частоти дихання у диханнях за хвилину
- Порівняння параметрів пошуку піків
- Порівняння окремих та комбінованих сигналів
- Обчислення комбінованих сигналів `magnitude_signal` та `pca_component`
- Формування фінального результату на основі PCA-компоненти
- Інтерактивна демонстрація всіх етапів обробки у Streamlit

## Як запустити проєкт "з нуля"

### 1. Встановлення інструментів

Для запуску проєкту потрібно встановити:

- Python 3.10 або новішу версію
- pip
- Git

---

### 2. Клонування репозиторію

```bash
git clone https://github.com/alicepohorliak/breathing-rate-accelerometer.git
cd breathing-rate-accelerometer
```

---

### 3. Створення віртуального середовища

```bash
python -m venv venv
```

---

### 4. Активація віртуального середовища

Для Windows:

```bash
venv\Scripts\activate
```

Для macOS / Linux:

```bash
source venv/bin/activate
```

---

### 5. Встановлення залежностей

```bash
pip install -r requirements.txt
```

---

### 6. Запуск інтерактивного інтерфейсу

```bash
streamlit run app.py
```

Після запуску у браузері відкриється інтерактивний інтерфейс для демонстрації поетапної обробки сигналів акселерометра.

---

### 7. Запуск консольної версії алгоритму

```bash
python main.py
```

Файл `main.py` виконує фінальний алгоритм у терміналі без вебінтерфейсу та зберігає результати у папки `results/` і `figures/`.

## Основні етапи обробки сигналу

1. Зчитування CSV-файлу з даними акселерометра
2. Перевірка основних колонок: `time[us]`, `acc_x[mg]`, `acc_y[mg]`, `acc_z[mg]`
3. Перетворення часу з мікросекунд у секунди
4. Побудова сирих сигналів по трьох осях акселерометра
5. Центрування сигналів для усунення базового зміщення
6. Ресемплінг сигналів до частоти 50 Гц для отримання рівномірного часового кроку
7. Смугова фільтрація у діапазоні 0.1–0.7 Гц
8. Пошук піків за параметрами `min_distance` та `prominence`
9. Розрахунок частоти дихання у диханнях за хвилину
10. Порівняння окремих осей і комбінованих сигналів та вибір PCA-компоненти для фінального результату

## Інструкція для користувача

1. Запустити застосунок командою:

```bash
streamlit run app.py
```

2. У бічній панелі вибрати джерело даних:
   - використати стандартний CSV-файл з папки `data/`;
   - або завантажити власний CSV-файл.

3. Переглянути перевірку вхідних даних:
   - кількість рядків;
   - кількість колонок;
   - тривалість запису;
   - наявність потрібних колонок;
   - пропущені значення.

4. Послідовно пройти етапи обробки сигналів:
   - побудова сирих сигналів;
   - центрування;
   - ресемплінг і фільтрація;
   - пошук піків;
   - розрахунок частоти дихання;
   - порівняння параметрів;
   - порівняння окремих і комбінованих сигналів;
   - фінальний результат алгоритму.

5. За потреби змінити параметри у бічній панелі:
   - частоту ресемплінгу;
   - нижню межу фільтра;
   - верхню межу фільтра;
   - сигнал для аналізу;
   - мінімальну відстань між піками;
   - prominence.

6. Переглянути фінальний результат:
   - обраний сигнал;
   - кількість знайдених піків;
   - частоту дихання у диханнях за хвилину;
   - фінальний графік із позначеними піками.

## Приклади результатів

У папці `figures/` зберігаються графіки, отримані під час виконання алгоритму:

- `raw_xyz.png` — сирі сигнали акселерометра по трьох осях;
- `centered_xyz.png` — центровані сигнали;
- `filtered_xyz.png` — відфільтровані сигнали;
- `basic_peaks.png` — результат базового пошуку піків;
- `params_comparison.png` — порівняння параметрів пошуку піків;
- `signals_comparison.png` — порівняння окремих і комбінованих сигналів;
- `final_peaks.png` — фінальний графік із позначеними піками.

## Проблеми і рішення

| Проблема                           | Рішення                                                                        |
| ---------------------------------- | ------------------------------------------------------------------------------ |
| Streamlit не запускається          | Активувати віртуальне середовище та встановити залежності з `requirements.txt` |
| CSV-файл не зчитується             | Перевірити, чи файл є в папці `data/`                                          |
| Відсутні потрібні колонки          | Перевірити наявність `time[us]`, `acc_x[mg]`, `acc_y[mg]`, `acc_z[mg]`         |
| Не відображаються графіки          | Перевірити встановлення `matplotlib` і `streamlit`                             |
| Знайдено забагато або замало піків | Змінити параметри `min_distance` або `prominence`                              |
